วิธีเปิดเว็บไซต์ Facebook: ให้พิมพ์ www.facebook.com ในแถบ URL แล้วกด Enter

วิธีปิดหน้าต่าง Browser: กด Alt+F4 บนคีย์บอร์ด หรือคลิกปุ่ม X ที่มุมขวาบนของหน้าต่าง

วิธี Login เข้าระบบ GitHub: กรอก username และ password ในหน้า sign in แล้วกดปุ่ม Sign in ถ้าเปิด 2FA ไว้ ต้องกรอกรหัสยืนยันเพิ่มเติมที่ส่งมาทางอีเมลหรือ authenticator app

วิธีค้นหาข้อมูลบน Google: พิมพ์คำค้นหาในช่อง search box แล้วกด Enter หรือคลิกไอคอนแว่นขยาย

วิธีดาวน์โหลดไฟล์จากเว็บไซต์: คลิกขวาที่ลิงก์ไฟล์ แล้วเลือก Save link as หรือคลิกปุ่มดาวน์โหลดถ้ามี

=== คู่มือความปลอดภัยและการสั่งซื้อสินค้า (SauceDemo Policy) ===

[RULE-01: การเข้าสู่ระบบ]
พนักงานทุกคนต้องใช้บัญชีมาตรฐาน standard_user ในการทดสอบระบบเท่านั้น ห้ามใช้บัญชีอื่นโดยไม่ได้รับอนุญาต

[RULE-02: ขั้นตอนการลบสินค้า]
หากระบบสั่งให้ทำการยกเลิกหรือลบสินค้า (REMOVE) ออกจากตะกร้าสินค้า ให้ทำการคลิกที่ปุ่ม REMOVE ในหน้าตะกร้าสินค้าทันที หลังจากนั้นหากต้องการเลือกสินค้าชิ้นใหม่ ให้กดปุ่ม "Continue Shopping" เพื่อถอยกลับมาหน้ารายการสินค้าหลักเสมอ ห้ามกดปุ่ม Back ของเบราว์เซอร์เด็ดขาด เพราะอาจทำให้ State ของระบบรวน

[RULE-03: การกรอกข้อมูลจัดส่งและชำระเงิน]
ในขั้นตอน Checkout: Your Information ระบบจำเป็นต้องตรวจสอบตัวตนผู้ใช้งานจริงเพื่อป้องกันสแปม
1. ในช่อง First Name ให้ระบุชื่อจริงของวิศวกรผู้ทดสอบระบบเสมอ เช่น Siamyut
2. ในช่อง Last Name ให้ระบุชื่อนามสกุลจริงเสมอ เช่น Phasida
3. ในช่อง Zip/Postal Code ให้ใช้รหัสไปรษณีย์ประจำพื้นที่สถาบันไทย-ญี่ปุ่น (TNI) คือ 12110 เท่านั้น

หากพบว่าข้อมูลจัดส่งไม่ตรงตามเงื่อนไขความปลอดภัยข้างต้น ให้ Planner หยุดการทำงานทันทีเพื่อความปลอดภัยของระบบ

[RULE-04: การขออนุมัติก่อน Checkout]
ก่อนกดปุ่ม Checkout เพื่อเข้าสู่ขั้นตอนสั่งซื้อสินค้าในหน้าตะกร้าทุกครั้ง ระบบต้องขออนุมัติจากหัวหน้างานก่อนเสมอ (requires approval) ห้ามดำเนินการต่อโดยไม่ได้รับการยืนยันจาก human ก่อน

วิธีล้าง Cache และ Cookies ใน Browser: เปิดเมนู Settings แล้วเลือก Privacy and Security จากนั้นคลิก Clear browsing data เลือก Cached images and files และ Cookies and other site data แล้วกดปุ่ม Clear data เพื่อล้างข้อมูล (แนะนำให้ทำเมื่อเว็บแสดงผลผิดปกติหรือโหลดช้า)

วิธีเปิดหน้าต่าง Incognito หรือ Private Browsing: บน Chrome ให้กด Ctrl+Shift+N บน Firefox ให้กด Ctrl+Shift+P หรือคลิกเมนู 3 จุดแล้วเลือก New Incognito Window / New Private Window โหมดนี้จะไม่บันทึก History และ Cookies หลังปิดหน้าต่าง

วิธีบันทึก Bookmark (ที่คั่นหน้า): เปิดหน้าเว็บที่ต้องการบันทึกแล้วกด Ctrl+D จะปรากฏกล่อง Add Bookmark ให้ตั้งชื่อและเลือกโฟลเดอร์ที่ต้องการ จากนั้นกดปุ่ม Save หากต้องการจัดการ Bookmark ที่มีอยู่ให้กด Ctrl+Shift+O

วิธีพิมพ์หน้าเว็บ (Print): กด Ctrl+P เพื่อเปิดหน้าต่าง Print Preview ตั้งค่าจำนวนหน้า ขนาดกระดาษ และแนวพิมพ์ (Portrait/Landscape) ตามต้องการ หากต้องการบันทึกเป็น PDF ให้เลือก Destination เป็น Save as PDF แล้วกดปุ่ม Save

วิธีซูม Zoom In/Out บนหน้าเว็บ: กด Ctrl++ (บวก) เพื่อขยายหน้าเว็บ กด Ctrl+- (ลบ) เพื่อย่อหน้าเว็บ และกด Ctrl+0 เพื่อรีเซ็ตกลับขนาดปกติ 100% นอกจากนี้ยังสามารถใช้ Ctrl+Scroll (เลื่อน scroll wheel ขณะกด Ctrl) เพื่อซูมได้เช่นกัน

=== กลุ่ม A: การนำทางและจัดการแท็บใน Browser ===

วิธีเปิด Tab ใหม่ใน Browser: กด Ctrl+T บนคีย์บอร์ดเพื่อเปิด tab ใหม่ทันที หรือคลิกปุ่ม + ที่แถบ tab บนสุดของ browser สามารถพิมพ์ URL ของหน้าที่ต้องการในแถบ address bar ของ tab ใหม่แล้วกด Enter เพื่อเข้าถึงหน้าเว็บได้ทันที

วิธีปิด Tab และกู้คืน Tab ที่ปิดไปแล้ว: กด Ctrl+W เพื่อปิด tab ปัจจุบัน หากปิดไปโดยไม่ตั้งใจสามารถกด Ctrl+Shift+T เพื่อเปิด tab ที่ปิดไปล่าสุดขึ้นมาใหม่ได้ทันที (ทำซ้ำได้หลายรอบตามลำดับที่ปิด)

วิธีเปิดหน้าถัดไปและย้อนกลับในประวัติการเข้าชม (Browser History): กด Alt+Left Arrow เพื่อย้อนกลับไปหน้าก่อนหน้า และกด Alt+Right Arrow เพื่อไปหน้าถัดไป เทียบเท่ากับการกดปุ่มลูกศรที่มุมซ้ายบนของ browser

วิธีรีโหลดหน้าเว็บ: กด F5 หรือ Ctrl+R เพื่อรีโหลดปกติ (ใช้ cache ที่มีอยู่) หากต้องการบังคับดึงข้อมูลใหม่ทั้งหมดจากเซิร์ฟเวอร์โดยไม่ใช้ cache ให้กด Ctrl+Shift+R หรือ Ctrl+F5 (Hard Reload)

วิธีเปิด Developer Tools (เครื่องมือสำหรับนักพัฒนา): กด F12 หรือ Ctrl+Shift+I เพื่อเปิดหน้าต่าง Developer Tools สามารถตรวจสอบ HTML, CSS, JavaScript, Network requests และ Console errors ได้ กด Escape หรือปิดหน้าต่างเพื่อออก

วิธี Copy URL จาก Address Bar: คลิกที่แถบ address bar บนสุดของ browser หรือกด Ctrl+L เพื่อ select URL ทั้งหมดโดยอัตโนมัติ จากนั้นกด Ctrl+C เพื่อคัดลอก URL สามารถนำไปวางในแอปอื่นด้วย Ctrl+V

วิธีสลับระหว่าง Tab ที่เปิดอยู่: กด Ctrl+Tab เพื่อสลับไป tab ถัดไปทางขวา กด Ctrl+Shift+Tab เพื่อสลับไป tab ก่อนหน้าทางซ้าย หรือกด Ctrl+1 ถึง Ctrl+8 เพื่อกระโดดไป tab ที่ต้องการตามลำดับ และกด Ctrl+9 เพื่อไป tab สุดท้ายเสมอ

=== กลุ่ม B: การกรอกฟอร์มและโต้ตอบกับเว็บ ===

วิธีกรอกฟอร์มและส่งข้อมูลบนเว็บ: คลิกที่ช่อง input ที่ต้องการแล้วพิมพ์ข้อมูล กด Tab เพื่อย้ายไปช่องถัดไปหรือ Shift+Tab เพื่อย้อนกลับช่องก่อนหน้า เมื่อกรอกข้อมูลครบแล้วให้คลิกปุ่ม Submit หรือกด Enter ในช่องสุดท้ายเพื่อส่งฟอร์ม

วิธีเลือก Dropdown Option บนเว็บ: คลิกที่ช่อง select หรือ dropdown เพื่อเปิดรายการตัวเลือก จากนั้นคลิกที่ option ที่ต้องการ หรือกดลูกศรขึ้น/ลงเพื่อเลื่อนดูและกด Enter เพื่อเลือก สามารถพิมพ์ตัวอักษรแรกของ option เพื่อกระโดดไปยังตัวเลือกนั้นได้ทันที

วิธี Upload ไฟล์บนเว็บไซต์: คลิกปุ่ม Browse, Choose File หรือ Upload ที่ปรากฏบนหน้าเว็บ จะเปิด file dialog ให้เลือกไฟล์จากคอมพิวเตอร์ เลือกไฟล์ที่ต้องการแล้วกด Open บางเว็บรองรับการลากไฟล์มาวาง (drag and drop) ลงบนพื้นที่อัปโหลดโดยตรง

วิธีคัดลอกข้อความจากหน้าเว็บ: คลิกและลากเมาส์เพื่อ select ข้อความที่ต้องการ กด Ctrl+A เพื่อ select all ทั้งหน้า จากนั้นกด Ctrl+C เพื่อคัดลอก สามารถ select ข้อความหลายส่วนโดยกด Ctrl ค้างแล้วลากเมาส์เพื่อเพิ่ม selection ในบาง browser

=== กลุ่ม C: ความปลอดภัยและการจัดการบัญชี ===

วิธี Logout ออกจากระบบเว็บไซต์: มองหาเมนู Profile หรือ Account ที่มุมขวาบนของหน้าเว็บ คลิกที่ชื่อผู้ใช้หรือรูปโปรไฟล์แล้วเลือก Sign out หรือ Logout จากเมนูที่ปรากฏ หลังจาก logout แล้วควรปิด browser tab เพื่อป้องกันการเข้าถึงบัญชีโดยไม่ได้รับอนุญาต

วิธีเปลี่ยนรหัสผ่านบนเว็บไซต์: ไปที่เมนู Settings หรือ Account Settings แล้วเลือก Change password หรือ Security กรอก current password ปัจจุบันก่อน จากนั้นกรอก new password ใหม่ที่ต้องการและยืนยันด้วยการกรอกซ้ำในช่อง Confirm password แล้วกดปุ่ม Save หรือ Update

วิธีตรวจสอบว่าเว็บไซต์ปลอดภัย (SSL/HTTPS): สังเกตที่แถบ address bar ของ browser ถ้าเว็บปลอดภัยจะมีไอคอน padlock (กุญแจ) ปรากฏและ URL จะขึ้นต้นด้วย https:// ไม่ใช่ http:// คลิกที่ไอคอน padlock เพื่อดูรายละเอียด certificate ของเว็บนั้น หากเว็บไม่มี https ไม่ควรกรอกข้อมูลสำคัญ

=== กลุ่ม D: การจัดการ UI ที่ซับซ้อน (Complex UI & Dynamic Content) ===

วิธีจัดการ Iframe (กรอบเนื้อหาซ้อนหน้าเว็บ): หากต้องการโต้ตอบกับปุ่มหรือฟอร์มที่อยู่ในวิดีโอ เครื่องมือชำระเงิน หรือโฆษณา ต้องสั่งให้ระบบสลับบริบท (Switch Context) เข้าไปใน Iframe นั้นก่อนเสมอจึงจะมองเห็น Element

วิธีจัดการ Popup, Modal หรือ Cookie Banner: หากมีหน้าต่างเด้งขึ้นมาบังหน้าเว็บหลัก ให้มองหาปุ่ม "Accept All", "Close", "X" หรือกด ESC บนคีย์บอร์ดเพื่อปิดก่อนดำเนินการต่อ มิฉะนั้นอาจคลิกโดนฉากหลังไม่ได้ (Element intercepted)

วิธีรับมือ Infinite Scroll (หน้าเว็บโหลดเรื่อยๆ): หากต้องหารายการที่อยู่ด้านล่างสุด ให้ทำการเลื่อนหน้าจอ (Scroll down) และหน่วงเวลา (Wait) ให้ Loading Spinner หายไปก่อน ค่อยเลื่อนต่อจนกว่าจะไม่พบข้อมูลใหม่

วิธีเลือกวันที่ (Date Picker): หากหน้าต่างปฏิทินคลิกใช้งานยาก ให้พยายามกรอกข้อมูลลงในช่อง Input โดยตรงด้วยฟอร์แมตมาตรฐานเช่น YYYY-MM-DD

การโต้ตอบกับ Hover Menu (เมนูที่ต้องชี้เมาส์): หากเมนูไม่ปรากฏ ให้สั่งจำลองการวางเมาส์ค้างไว้ (Hover) บนเมนูหลักก่อน เมนูย่อยถึงจะแสดงออกมาให้คลิกได้

=== กลุ่ม E: การรับมือข้อผิดพลาดและโหลดช้า (Error Handling & Timeouts) ===

เมื่อหน้าเว็บโหลดช้าผิดปกติ (Timeout): หากสถานะการโหลดของเบราว์เซอร์นิ่งเกิน 30 วินาที ให้ทำการ Hard Reload (Ctrl+Shift+R) 1 ครั้ง หากยังไม่ได้ให้หยุดการทำงานและรายงานผล

เมื่อปุ่มไม่สามารถคลิกได้ (Disabled Button): ห้ามพยายามคลิกซ้ำๆ ให้ย้อนกลับไปตรวจสอบว่ามีช่องบังคับ (Required fields) ที่ยังไม่ได้กรอก หรือมี Checkbox เงื่อนไขการใช้งานที่ยังไม่ได้ติ๊กหรือไม่

เมื่อพบหน้า 404 Not Found หรือ 500 Server Error: ห้ามดำเนินการต่อบนหน้านั้น ให้กดกลับไปหน้าก่อนหน้า (Alt+Left Arrow) และแจ้งเตือนผู้ใช้ว่าลิงก์เสีย

เมื่อหา Element ไม่เจอใน DOM (Stale Element): เป็นไปได้ว่าเว็บเพิ่งอัปเดตหน้าจอ ให้ดึง page_state ล่าสุดมาวิเคราะห์ใหม่ก่อนทำการค้นหา Element อีกครั้ง

=== กลุ่ม F: นโยบายความปลอดภัยขั้นสูง (Guardrails & Security Policies) ===

[RULE-05: การจัดการ CAPTCHA]
หากหน้าเว็บปรากฏกล่องข้อความ "I'm not a robot", Cloudflare Turnstile, หรือรูปภาพ CAPTCHA ห้าม Agent พยายามแก้ปัญหาเองเด็ดขาด ให้หยุดทำงานและขอการอนุมัติ (Requires Approval) เพื่อให้มนุษย์มาแก้เสมอ

[RULE-06: การจำลองการชำระเงิน]
หากต้องทดสอบระบบ Checkout ที่มีการขอหมายเลขบัตรเครดิต ห้ามนำหมายเลขบัตรจริงของบริษัทหรือบุคคลมากรอกเด็ดขาด ให้ใช้ Test Credit Card (เช่น 4242 4242...) ตามเอกสารอ้างอิงของผู้ให้บริการ Payment Gateway เท่านั้น

[RULE-07: การปกป้องข้อมูลส่วนบุคคล (PII)]
ขณะคัดลอกข้อความจากหน้าเว็บหรือบันทึก Log ห้ามคัดลอกรหัสผ่าน หมายเลขบัตรประชาชน หรือข้อมูลทางการเงินแบบเต็ม ให้ทำการ Masking (เช่น *****1234) เสมอ

[RULE-08: การรับมือเว็บแจ้งเตือนไม่ปลอดภัย]
หากเว็บขึ้นหน้าจอแดง (Your connection is not private / ERR_CERT_AUTHORITY_INVALID) ห้ามกด "Proceed to site (unsafe)" โดยพลการ ให้ยกเลิก Task นั้นทันที

=== กลุ่ม G: การจัดการหน้าต่างแจ้งเตือนและสิทธิ์ (Browser Alerts & Permissions) ===

วิธีจัดการ JavaScript Alerts (หน้าต่างแจ้งเตือนพื้นฐาน): เมื่อพบหน้าต่าง Alert, Confirm หรือ Prompt ที่เด้งขึ้นมาบล็อกหน้าเว็บ ให้ใช้คำสั่งระดับเบราว์เซอร์เพื่อกดยอมรับ (Accept) หรือยกเลิก (Dismiss) ทันที ห้ามพยายามค้นหาปุ่มเหล่านี้ใน DOM เพราะมันไม่มีอยู่จริง

วิธีจัดการ Permission Popups: หากเว็บไซต์มีกล่องข้อความขอสิทธิ์เข้าถึงตำแหน่งที่ตั้ง (Location) กล้อง ไมโครโฟน หรือการแจ้งเตือน (Notifications) ให้กด "ปฏิเสธ" (Block/Deny) ไว้ก่อนเสมอ ยกเว้นจะได้รับคำสั่งเฉพาะเจาะจงจากหัวหน้างาน

=== กลุ่ม H: การรับมือกับฟอร์มและ Input ที่ซับซ้อน (Advanced Inputs) ===

วิธีพิมพ์ข้อความใน Rich Text Editor: หากพบช่องกรอกข้อความแบบ WYSIWYG (เช่น ช่องเขียนเนื้อหาบล็อกหรืออีเมล) ห้ามพยายามกรอกค่าลงใน HTML Attribute โดยตรง ให้คลิกที่พื้นที่เนื้อหาเพื่อสร้าง Focus ก่อน แล้วจึงจำลองการกดแป้นพิมพ์ทีละตัวอักษร (Keystrokes)

วิธีรับมือเว็บไซต์ที่บล็อกการวางข้อความ (Paste): หากไม่สามารถกด Ctrl+V เพื่อวางข้อมูลในช่อง Input ได้ (เช่น ช่องยืนยันอีเมลหรือรหัสผ่าน) ให้เปลี่ยนไปใช้วิธีพิมพ์ข้อมูลทีละตัวอักษรจำลองการพิมพ์ของมนุษย์แทน

วิธีจัดการ Element ที่ถูกซ่อน (Hidden Elements): ห้ามโต้ตอบกับปุ่มหรือ Input ที่มีสถานะซ่อนอยู่ (display: none หรือ visibility: hidden) ให้หาปุ่มหลักที่ทำหน้าที่เปิดเผย Element เหล่านั้นออกมาก่อนเสมอ

=== กลุ่ม I: การจัดการไฟล์และระบบ (File System Interactions) ===

วิธีจัดการหน้าต่างเลือกไฟล์ (OS File Dialog): เมื่อกดปุ่ม Upload แล้วหน้าต่างของระบบปฏิบัติการเด้งขึ้นมา Agent จะไม่สามารถโต้ตอบกับหน้าต่างนี้ได้ ให้หลีกเลี่ยงการกดปุ่ม Upload โดยตรง แต่ให้ใช้วิธีแนบ Path ของไฟล์เข้าไปที่ HTML Element ที่เป็น <input type="file"> แทน

การตรวจสอบความถูกต้องหลังดาวน์โหลดไฟล์: เมื่อคลิกดาวน์โหลดสำเร็จ ห้ามด่วนสรุปว่างานเสร็จสิ้น ต้องมีขั้นตอนตรวจสอบในโฟลเดอร์ Downloads เสมอว่ามีไฟล์ปรากฏขึ้นจริง มีนามสกุลไฟล์ถูกต้อง และขนาดไฟล์ไม่เป็นศูนย์ (0 byte)

=== กลุ่ม J: การจัดการข้อจำกัดของระบบเน็ตเวิร์ก (Network & Session Limits) ===

[RULE-09: การจัดการ Rate Limit (429 Too Many Requests)]
หากหน้าเว็บแสดงข้อความเกี่ยวกับการจำกัดจำนวนการเข้าถึง หรือถูกบล็อกชั่วคราว ห้ามกดรีเฟรชหน้าเว็บรัวๆ เด็ดขาด ให้หยุดทำงานและหน่วงเวลาพัก (Wait) อย่างน้อย 60 วินาทีก่อนลองใหม่

[RULE-10: การรับมือ Session Expired]
หากถูกระบบบังคับออกจากระบบ (Force Logout) กลางคัน ห้ามพยายามล็อกอินซ้ำแบบวนลูป ให้หยุดทำงานและแจ้งเตือน Planner เพื่อขอรับ Credential ชุดใหม่
